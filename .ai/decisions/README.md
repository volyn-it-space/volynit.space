# Decision Memory

## Purpose

This folder stores durable repository decisions.

Use it for rules that should matter to future developers and agents, not for temporary notes.

Examples:
- page structure rules
- SEO policy
- styling policy
- Tailwind usage policy
- content management decisions
- agent workflow rules

## What Does Not Belong Here

Do not use this folder for:
- task notes
- bug logs
- chat summaries
- one-off experiments
- temporary status updates

## How To Use It

- keep one durable decision per file
- keep the index current
- prefer concise, actionable records
- update an existing decision instead of creating duplicates when possible
